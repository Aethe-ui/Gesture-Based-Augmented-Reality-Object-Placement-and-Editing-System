# Gesture-Based AR Block Builder

Python desktop AR prototype using OpenCV + MediaPipe + NumPy.

## Setup

1. Use Python 3.10+.
2. Install dependencies:

```bash
pip install opencv-python mediapipe numpy
```

3. Make sure `hand_landmarker.task` is present at the project root.
4. Generate an ArUco marker image:

```bash
python3 generate_marker.py
```

5. Run the app:

```bash
python3 main.py
```

## Controls

- `M`: cycle interaction mode (`PLACE` -> `MOVE` -> `DELETE`)
- `C`: cycle placement color
- `S`: save scene to `scene.json`
- `E`: export current blocks to `export.obj`
- `Q`: quit
- Undo: `Ctrl+Z` (fallbacks: `Z`, `U`)
- Redo: `Ctrl+Y` (fallbacks: `Y`, `R`)

## Interaction model

- Hold the ArUco marker (`DICT_6X6_250`, ID `0`) in view for marker-based tracking.
- If marker is not visible, app falls back to a virtual camera pose.
- Pinch (thumb tip + index tip) to interact:
  - `PLACE`: place/stack blocks on grid
  - `MOVE`: pick top block in column and drag
  - `DELETE`: remove top block in column

## Files generated by the app

- `scene.json`: saved block scene (`S`)
- `export.obj`: Wavefront OBJ export (`E`)
- `marker_id0.png`: generated marker image

## Troubleshooting

- **No camera feed**
  - Check macOS camera permission for Terminal/IDE.
  - Close other apps using the camera.
- **Hand tracking not working**
  - Verify `hand_landmarker.task` exists and is readable.
  - Improve lighting and keep hand fully in frame.
- **Marker tracking unstable**
  - Use a flat, non-glossy printed marker.
  - Keep marker fully visible and avoid motion blur.
- **Undo/redo shortcut conflict on macOS**
  - Use fallback keys `Z/U` for undo and `Y/R` for redo.

## Pre-release checklist

- Confirm `python3 -m unittest test_logic.py` passes.
- Run `python3 main.py` and verify:
  - mode switching (`M`), color cycling (`C`), save (`S`), export (`E`), undo/redo.
  - both `MARKER` and `VIRTUAL` tracking paths.
- Verify persistence by saving, quitting, and relaunching.
- Verify `export.obj` opens correctly in Blender/MeshLab.
- Ensure `hand_landmarker.task` is available for the target environment.
