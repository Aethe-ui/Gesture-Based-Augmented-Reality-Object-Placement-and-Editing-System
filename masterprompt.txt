## PROJECT SUMMARY

We are building a Python desktop application that:
- Opens a webcam feed
- Detects hand landmarks in real time (MediaPipe)
- Reads a printed ArUco marker to estimate real camera pose
- Uses that pose to ray-cast hand gestures onto a virtual ground plane
- Lets the user place, move, and delete 3D blocks in AR using a pinch gesture
- Renders solid 3D cubes projected onto the camera frame using OpenCV

Tech stack: Python 3 · OpenCV · MediaPipe Tasks API · NumPy
No Unity, no WebXR, no external game engines.

Known issues in current build:
1. No jitter filtering on ray-cast ground coordinates → shaky block placement
2. Camera feed is unmirrored (correct for ArUco) but feels unnatural for webcam use
3. No block persistence — scene resets on every run
4. Only ArUco marker ID 0 (DICT_6X6_250) is supported
5. Camera intrinsics are approximate (focal = frame width, zero distortion)
6. Painter's algorithm can flicker on steep camera angles

## PHASE 3 — 3D Math & AR Rendering
## ═══════════════════════════════════════

GOAL: Implement ray casting, 3D→2D projection, and cube rendering.
      The pinch midpoint should now map to a point on the ground plane.

DELIVERABLES:
1. ar_math.py (complete)
   - get_camera_matrix(w, h) — as before
   - project_point_3d_to_2d(point_3d, rvec, tvec, K) → (px, py) tuple
     Uses cv2.projectPoints
   - ray_cast_to_ground(uv, K, rvec, tvec)
     Cast ray from camera through screen point uv=(u,v)
     Intersect with world plane Z=0
     Return (x, y, 0) or None if parallel/behind camera
     Algorithm:
       1. Normalized ray dir in camera frame: x_c=(u-cx)/fx, y_c=(v-cy)/fy, z_c=1
       2. Ray to world: R_inv = R.T; origin = -R_inv @ tvec; dir = R_inv @ ray_cam
       3. Intersect Z=0: lambda = -origin.z / dir.z
       4. Point = origin + dir * lambda
   - draw_cube(img, center_3d, size, rvec, tvec, K, color)
     8 corners → project each → sort 6 faces by depth (Painter's algorithm) →
     fillPoly each face (slightly darkened), polylines border
   - draw_grid(img, rvec, tvec, K) — as before

2. main.py update
   - After pinch detected: call ray_cast_to_ground(center, K, rvec, tvec)
   - Draw a yellow dot at the projected ground point
   - Draw a test cube at (0, 0, 0) always — so we can verify 3D rendering works
   - Remove test cube once rendering is confirmed

VERIFICATION TEST:
- A cube should appear anchored at the origin of the marker
- Moving the camera around it should make it look correctly 3D
- The yellow dot should appear where the hand points on the ground

## PHASE 4 — Block System & Interaction Modes
## ═══════════════════════════════════════

GOAL: Full block manipulation — place, move, delete — with mode switching.

DELIVERABLES:
1. block_manager.py (complete)
   - BlockManager class with self.blocks = []
   - Each block: {'pos': (x, y, z), 'color': (B, G, R)}
   - snap_to_grid(x, y, z) → snaps each axis to nearest GRID_SPACING multiple
   - add_block(x, y, z, color) → snaps, checks duplicate, appends, returns bool
   - get_block_at(x, y, z, tolerance=None)
     With tolerance: distance-based nearest-block search
     Without: strict grid match
     Returns (index, block) or (-1, None)
   - remove_block(x, y, z) → find + delete, returns bool
   - move_block(index, new_x, new_y, new_z) → snap, collision check, update
   - get_blocks() → returns self.blocks

2. main.py (complete — final version)
   Three modes cycled with M key:
   - PLACE (green): Pinch → ray-cast → stack new block on top of column
     Stacking: scan blocks at same (snap_x, snap_y), find highest z, add at z+BLOCK_SIZE
     Cooldown: 0.5s between placements
   - MOVE (blue): First pinch → select nearest block (tolerance=BLOCK_SIZE)
                  Held pinch → drag block to new ground position (keep same z)
                  Release → deselect
   - DELETE (red): Pinch → remove nearest block (tolerance=BLOCK_SIZE*0.8)
                   Cooldown: 0.5s

   UI overlay rectangle (top-left):
   - FPS counter
   - Current mode + "(Press M)"
   - Tracking mode: MARKER or VIRTUAL

   Selected block renders in YELLOW

VERIFICATION TEST: Run python main.py.
- Place 3 cubes, verify they stack
- Switch to MOVE, drag a cube
- Switch to DELETE, remove a cube
- All blocks should remain stable across mode switches
